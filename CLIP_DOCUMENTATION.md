# Clip — Документация

## Обзор

**Clip** — мобильное приложение для Android (и iOS) на базе Expo + React Native. Личный архив цитат и мыслей с механикой ежедневного возврата. Всё хранится локально на устройстве — никакого бэкенда.

## Экраны

### OnboardingScreen (`app/onboarding.tsx`)

Показывается при первом запуске. Сохраняет флаг `onboardingDone = true` в настройках через AsyncStorage.

**Шаг 1 — Первая цитата**
- Заголовок: "Сохраняй лучшее. Вспоминай каждый день."
- Открытое поле ввода текста
- Кнопка "Сохранить и продолжить" → сохраняет цитату и переходит дальше
- Кнопка "Позже" → пропускает шаг без сохранения

**Шаг 2 — Share Intent**
- Визуальная инструкция из 3 шагов: выдели → поделись → Clip
- Кнопка "Понял, продолжить"

**Шаг 3 — Время уведомлений**
- Три кнопки: "Утром (8:00)", "Вечером (20:00)", "Не сейчас"
- При выборе времени: запрашивает разрешение, планирует ежедневный push
- Переходит на главный экран

---

### HomeScreen (`app/(tabs)/index.tsx`)

Главный экран с двумя зонами.

**Верхняя зона — дневные карточки**
- Дата сегодня + заголовок "Сегодняшние открытия"
- 3 случайные карточки из архива (фиксируются на день через `getDailyCards()`)
- При пустом архиве — empty state с кнопкой "Сохранить первую цитату"

**Нижняя зона**
- Кнопка "✦ Удиви меня" → показывает случайную карточку в анимированном модальном окне
- Быстрое поле ввода для добавления цитаты прямо с главного экрана
- FAB "+" в правом нижнем углу → открывает AddClipScreen

---

### ArchiveScreen (`app/(tabs)/archive.tsx`)

**Поиск** — строка поиска фильтрует в реальном времени по тексту и тегам.

**Фильтр по тегам** — горизонтальный скролл: "Все" + по одному тегу на каждый существующий.

**Список** — FlatList всех цитат, новые сверху. Компонент `ClipCard` в компактном режиме.

**Long press** → Alert с подтверждением удаления.

---

### AddClipScreen (`app/add.tsx`)

Открывается как modal снизу.

- Поле текста цитаты (обязательное, multiline)
- Компонент TagPicker для выбора/создания тегов
- Если источник не "manual" (приходит из Share Intent) — отображается бейдж источника
- При достижении лимита 100 цитат — показывает баннер и блокирует сохранение
- Кнопки: "✕" (закрыть) слева, "Сохранить" (янтарная) справа

---

### ClipDetailScreen (`app/clip/[id].tsx`)

Открывается по тапу на карточку.

- Полный текст цитаты крупным шрифтом с янтарной полосой-акцентом
- Метаданные: источник, дата
- Редактирование тегов (inline через TagPicker)
- Кнопка "Поделиться" → нативный Share
- Кнопка "Удалить" (с подтверждением Alert)

## Storage Layer (`src/storage/clips.ts`)

Все функции асинхронные, работают с AsyncStorage.

| Функция | Возвращает | Описание |
|---|---|---|
| `getAllClips()` | `Clip[]` | Все цитаты, новые сверху |
| `saveClip(clip)` | `Clip` | Сохраняет новую цитату, обновляет streak |
| `deleteClip(id)` | `void` | Удаляет по ID |
| `updateClip(id, changes)` | `void` | Обновляет поля цитаты |
| `getDailyCards()` | `Clip[]` | 3 случайных цитаты (фиксируются на день) |
| `getRandomClip()` | `Clip \| null` | Одна случайная цитата |
| `getAllTags()` | `string[]` | Все уникальные теги |
| `getStreak()` | `Streak` | Текущий streak |
| `getSettings()` | `Settings` | Настройки приложения |
| `saveSettings(s)` | `void` | Сохраняет настройки |

**Константа:** `FREE_LIMIT = 100`

**Структура Clip:**
```typescript
{
  id: string;          // Auto-generated
  text: string;        // Текст цитаты
  imageUri: null;      // Зарезервировано для скриншотов
  source: string;      // 'manual' | название приложения
  tags: string[];      // Теги
  createdAt: string;   // ISO строка
}
```

**AsyncStorage ключи:**
- `@clip:clips` — массив всех цитат
- `@clip:streak` — данные streak
- `@clip:settings` — настройки (notificationHour, onboardingDone)
- `@clip:daily_cards` — ID карточек на сегодня
- `@clip:daily_date` — дата последней генерации дневных карточек

## Push-уведомления (`src/notifications/digest.ts`)

| Функция | Описание |
|---|---|
| `requestNotificationPermission()` | Запрашивает разрешение системы |
| `scheduleDailyDigest(hour)` | Планирует ежедневный push в указанный час |
| `cancelDailyDigest()` | Отменяет все запланированные уведомления |

**Формат уведомления:**
- Заголовок: "Твои открытия"
- Текст: первые 60 символов случайной цитаты + "..."

На web push-уведомления не поддерживаются (автоматически отключены через `Platform.OS === 'web'`).

## Навигация

Expo Router с файловой маршрутизацией.

```
/ (tabs)
  ├── index     → Главный экран (Сегодня)
  └── archive   → Архив

/onboarding    → Онбординг (redirect при первом запуске)
/add           → Добавление цитаты (modal)
/clip/[id]     → Детальный просмотр
```

Tab bar поддерживает NativeTabs с Liquid Glass на iOS 26+ и классические вкладки с BlurView на остальных платформах.

## Проверка работы

### Ручное тестирование

1. **Онбординг:** При первом запуске должен показываться онбординг. После прохождения — редирект на главный экран. Повторный запуск — сразу главный экран.

2. **Добавление цитаты:**
   - Нажать "+" (FAB) или ввести текст в быстрое поле внизу
   - Ввести текст → добавить теги → нажать "Сохранить"
   - Цитата должна появиться в архиве

3. **Главный экран:**
   - После добавления 1+ цитат должны появиться дневные карточки
   - Кнопка "Удиви меня" → открывает случайную цитату

4. **Архив:**
   - Поиск по тексту работает в реальном времени
   - Фильтр по тегам фильтрует список
   - Long press на карточке → подтверждение удаления

5. **Детальный просмотр:**
   - Тап на карточку → полный текст
   - Кнопка "Редактировать" → TagPicker inline
   - Кнопка "Поделиться" → нативный Share
   - Кнопка "Удалить" → подтверждение → возврат в архив

6. **Лимит:** При 100+ цитатах добавление блокируется с баннером.

7. **Уведомления:** На реальном устройстве (Android/iOS) — выбор времени в онбординге запрашивает разрешение и планирует ежедневный push.

## Share Intent

Приложение поддерживает захват текста из других приложений через нативный механизм Share.

### Как это работает

Пользователь выделяет текст в Telegram, браузере или любом другом приложении, нажимает «Поделиться» и выбирает Clip — приложение открывается с предзаполненным текстом на экране добавления.

### Реализация

| Файл | Роль |
|---|---|
| `package.json` | `expo-share-intent@5.1.1` — совместима с Expo SDK 54 |
| `app.json` | Плагин `expo-share-intent` с фильтрами для текста и URL |
| `app/_layout.tsx` | `ShareIntentProvider` оборачивает дерево навигации; компонент `ShareIntentHandler` следит за входящими интентами |

### Конфигурация плагина (`app.json`)

```json
[
  "expo-share-intent",
  {
    "iosActivationRules": {
      "NSExtensionActivationSupportsText": true,
      "NSExtensionActivationSupportsWebURLWithMaxCount": 1
    },
    "androidIntentFilters": ["text/*"]
  }
]
```

- **iOS** — поддерживает текст (`NSExtensionActivationSupportsText`) и веб-URL (лимит 1)
- **Android** — фильтр `text/*` захватывает любой текстовый контент

### Логика `ShareIntentHandler`

```tsx
// Внутри провайдера, следит за hasShareIntent
const { hasShareIntent, shareIntent, resetShareIntent } = useShareIntentContext();

useEffect(() => {
  if (!hasShareIntent) return;
  const text = shareIntent?.text ?? shareIntent?.webUrl ?? null;
  if (!text) return;

  router.push({
    pathname: "/add",
    params: {
      sharedText: text,
      source: shareIntent?.meta?.title ?? "share",
    },
  });
  resetShareIntent(); // очищает интент после навигации
}, [hasShareIntent, shareIntent]);
```

- `shareIntent.text` — выделенный текст
- `shareIntent.webUrl` — URL если поделились ссылкой
- `shareIntent.meta?.title` — название приложения-источника (например, «Telegram»)
- После навигации `resetShareIntent()` сбрасывает состояние

### Ограничения

- **Expo Go / Web**: Share Intent отключается автоматически через `disabled: Platform.OS === 'web'`. Функция доступна только в нативной production-сборке.
- Для тестирования на симуляторе нужен `expo prebuild` + `expo run:android` / `expo run:ios`.
- В production-сборке для iOS используется Replit Expo Launch (App Store).

## Ограничения MVP

- Share Intent требует нативной сборки — в Expo Go и web-preview недоступен (отключён автоматически)
- Push-уведомления работают только на реальном устройстве (не в Expo Go на web)
- Только тёмная тема — светлого режима нет
- Лимит 100 цитат в бесплатной версии
- Скриншоты (`imageUri`) зарезервированы, но не реализованы в MVP
